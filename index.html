Problem Statment : Network Forensics


In the wake of increasing cyber threats and security breaches, network forensics has become an essential component in the field of cybersecurity. Packet capture analysis plays a critical role in network forensics as it provides valuable insights into network traffic, which can help in identifying malicious activities and potential security vulnerabilities.

Packet capture analysis (pcap) can uncover a wide range of vulnerabilities and cyber threats, including but not limited to:


  1. Plaintext Passwords: password authentication happening over plaintext.
  2. Intrusions and Breaches: port scanning attempt.
  3. PII Leak: Personally Identifiable Information (PII) like SSN, Credit Card numbers etc.
  4. Malware Infections: command and control (C2) traffic that is indicative of malware infections.
  5. Protocol Abnormalities: Deviations from standard protocol behavior.
  6. Network Misconfigurations: Use of unsecure protocols.
  7. Data Exfiltration: Large, unexplained data transfers.
  8. Denial of Service (DoS) or Distributed Denial of Service (DDoS) Attacks: A sudden surge in traffic from a single source or multiple sources.

The task at hand for this hackathon is to develop an innovative solution that can simplify the process of packet capture analysis for network forensics. The solution should be able to efficiently process large volumes of network traffic data, filter out noise, identify anomalies, and provide actionable insights for forensic analysis.
